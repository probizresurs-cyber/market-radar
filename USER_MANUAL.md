# 1. Дашборд руководителя

## 1.1. Что это и зачем
Главный экран для собственника бизнеса — единая сводка по всем основным метрикам компании, конкурентам, контенту и SEO в одном месте.

## 1.2. Как зайти
- **Сайдбар:** раздел **ОБЗОР** → **Дашборд руководителя**
- **Прямой URL:** `/owner-dashboard` (открывается отдельной вкладкой, удобно держать постоянно открытым)

## 1.3. Главный экран
🖼 **[СКРИН: Дашборд руководителя — общий вид]**

При открытии вы видите четыре крупных метрики в виде ярких карточек: **Конкуренты**, **Угрозы**, **Ваша доля**, **Ваш балл**. Под ними — таб-бар с разделами: Обзор, Действия, Репутация, Финансы, Команда/Найм, Компания, Конкуренты, Целевая аудитория, CJM.

Сверху — **карточка «Быстрый анализ любого сайта»**: можно ввести URL конкурента или партнёра и получить быстрый отчёт за 30 секунд без перехода в основное приложение.

Ниже — **карточка «Готовность модулей»** с цветными индикаторами: какие модули анализа запущены (ЦА, СММ, Customer Journey, Бенчмарки, Контент-завод, Брендбук).

Ещё ниже — **карточка SEO-позиций (Keys.so)**: общее количество ключей, позиции в ТОП-3 / ТОП-10, дневной трафик, разделение Яндекс / Google.

## 1.4. Сценарии использования

**Сценарий 1 — Утренняя сводка:**
1. Откройте дашборд руководителя
2. Посмотрите на 4 главных метрики — сразу видно состояние бизнеса
3. Если **«Угрозы: 2 активных»** — кликните карточку, откроется список угроз с пояснениями
4. Если **«Ваша доля»** упала с прошлой недели — переходите в раздел «Конкуренты» через таб-бар

**Сценарий 2 — Быстрый анализ конкурента:**
1. В верхней карточке «Быстрый анализ» введите URL: `competitor.ru`
2. Нажмите **Анализ**
3. Через 30 секунд получите Score, топ-4 категории, первое действие для улучшения
4. Если хотите подробнее — нажмите **«Открыть полный отчёт»** (только в private режиме)

**Сценарий 3 — Переход в подмодули:**
1. На дашборде кликните таб **Целевая аудитория** в таб-баре
2. Откроется встроенный мини-обзор ЦА
3. Если нужны детали — кнопка **«Открыть полный анализ»** ведёт на `/?nav=ta-dashboard`

## 1.5. Ключевые функции
- **4 главных KPI** одной строкой (Конкуренты / Угрозы / Доля рынка / Ваш балл из 100)
- **Быстрый анализ любого URL** без выхода с дашборда
- **Готовность 6 модулей** с прямыми переходами по клику
- **Keys.so SEO-метрики** — реальные данные по позициям Яндекс/Google
- **Конкурентный ландшафт** — список конкурентов с прогресс-барами
- **«Угрозы и возможности»** — карточки с пометками «ВНИМАНИЕ»
- **TabBar навигация** — клик по любому табу открывает соответствующий модуль в основном приложении

## 1.6. Интеграции
- **Keys.so** — реальные SEO-позиции и трафик
- **Google Places / Яндекс.Карты / 2GIS** — для рейтингов компаний
- **Claude AI** — для суммаризации модульных данных
- **Внутренние модули платформы** — все KPI берутся из результатов модулей (ЦА, СММ, Конкуренты)

## 1.7. Роли и права
- **Владелец / Admin:** видит все метрики, включая финансовые
- **Manager:** видит все, кроме финансов и команды
- **Viewer (приглашённый):** только KPI и Конкурентный ландшафт без раскрытых угроз

## 1.8. Типичные кейсы

**Кейс 1 — Стоматология «Менделеев»:** владелица каждое утро открывает дашборд за 30 секунд видит — Score 56/100, угрозы: «Рост сетевых конкурентов», доля 86% (лидер ниши). Принимает решение: запустить срочный контент по конкурентам в Контент-заводе.

**Кейс 2 — Производственная компания AllPlans:** Score упал с 32 до 9. На дашборде красная метрика → клик → инсайт «Сайт не индексируется в Google». Переход в SEO-статьи → запуск новых публикаций.

**Кейс 3 — Партнёр-агентство:** через «Быстрый анализ» проверяет 5 потенциальных клиентов в день. Сохраняет Score и топ-проблему каждого, готовит коммерческие предложения.

# 2. Новый анализ компании

## 2.1. Что это и зачем
Запуск полного анализа компании по URL — система парсит сайт, соцсети, отзывы на картах и формирует развёрнутый отчёт за 60–90 секунд.

## 2.2. Как зайти
- **Сайдбар:** раздел **АНАЛИТИКА** → **Моя компания** → **Новый анализ**

## 2.3. Главный экран
🖼 **[СКРИН: Multi-step Wizard, шаг 1 — ввод URL]**

Открывается визард с 4-5 шагами:
1. **URL компании** — поле ввода + кнопка «Далее»
2. **Какие модули запустить** — 4 чекбокс-карточки: «Анализ ЦА», «СММ-анализ», «Конкуренты», «Отзывы». У каждой — оценка времени (≈3 мин / 5 мин / 7 мин / 10 мин)
3. **Дополнительные данные для СММ** (если выбран модуль СММ) — поля для VK / Telegram / Instagram ссылок
4. **URL конкурентов** (если выбран модуль Конкуренты) — до 3 ссылок
5. **Подтверждение и запуск**

## 2.4. Сценарии использования

**Сценарий 1 — Базовый анализ только моей компании:**
1. Введите URL: `mycompany.ru` → **Далее**
2. На шаге модулей **снимите все чекбоксы** → **Далее**
3. Нажмите **«Запустить анализ»**
4. Через 60-90 секунд получите дашборд компании

**Сценарий 2 — Комплексный анализ с конкурентами:**
1. Введите URL компании → **Далее**
2. Включите все 4 модуля → **Далее**
3. На шаге СММ вставьте ссылки на ваши соцсети
4. На шаге конкурентов добавьте 2-3 URL
5. **Запустить** — модули анализа пойдут параллельно (3-10 минут)

**Сценарий 3 — Только конкурентный анализ:**
1. URL вашей компании → **Далее**
2. Снимите все кроме «Конкуренты»
3. Добавьте 3 URL конкурентов
4. Через ~7 минут получите дашборд + список конкурентов с сравнением

## 2.5. Ключевые функции
- **Визард с условными шагами** — модули СММ и Конкуренты добавляют свои шаги, остальные пропускаются
- **Параллельный запуск модулей** — экономия времени
- **Оценка времени** — видите сколько займёт перед запуском
- **Прогресс-бар** во время анализа с подсказками «Парсим сайт... → Анализируем соцсети... → Готовим отчёт»

## 2.6. Интеграции
- **Парсер сайта** (Playwright) — извлекает текст, мета-теги, изображения, vacancies
- **Claude Sonnet 4.6** — основной AI для анализа
- **Google Places / Яндекс.Карты / 2GIS** — для подтягивания рейтингов
- **DaData** — реквизиты компании РФ
- **HH.ru** — данные о вакансиях для оценки команды
- **PageSpeed Insights** — производительность сайта

## 2.7. Роли и права
- **Владелец:** может запускать любые анализы
- **Manager:** может запускать, но не редактировать результаты других пользователей
- **Лимиты:** 5 анализов в день на тарифе Старт, безлимит на Бизнес

## 2.8. Типичные кейсы

**Кейс 1 — Аудит перед редизайном сайта:** запустить полный анализ перед заказом нового сайта, получить список из 20+ конкретных проблем для брифа дизайнеру.

**Кейс 2 — Quarterly review:** раз в квартал прогонять компанию через анализ, сравнивать Score с прошлым (видно в «Предыдущие анализы»).

**Кейс 3 — Due diligence перед инвестициями:** проанализировать целевую компанию + её 3-х конкурентов за 15 минут до встречи.

# 3. Дашборд компании

## 3.1. Что это и зачем
Развёрнутый отчёт по компании: оценки по 6 категориям, SEO, соцсети, отзывы, рекомендации — всё в одном представлении.

## 3.2. Как зайти
- **Сайдбар:** **АНАЛИТИКА** → **Моя компания** → **Дашборд**
- Откроется автоматически после завершения «Нового анализа»

## 3.3. Главный экран
🖼 **[СКРИН: Дашборд компании — Score + категории]**

В шапке — **общий Score** (число из 100), название компании, дата анализа.

Ниже — **6 категорий с прогресс-барами**: SEO, Контент, Соцсети, Карты/Отзывы, Технологии, Команда. Каждая — кликабельная, раскрывает детали.

Далее — **«Главный вывод»** — AI-резюме на 3-5 предложений: что делать первым.

Ниже — **рекомендации**, разделённые по приоритету (Срочно / Важно / На потом).

В правой колонке — **живые рейтинги с карт**: Google, Яндекс, 2GIS.

## 3.4. Сценарии использования

**Сценарий 1 — Первый просмотр после анализа:**
1. Анализ завершился → автоматический переход на дашборд
2. Посмотрите общий Score
3. Прочитайте «Главный вывод»
4. Прокрутите вниз до рекомендаций → начните с раздела «Срочно»

**Сценарий 2 — Обновить данные после изменений:**
1. После правок на сайте откройте дашборд
2. В правом верхнем углу — кнопка **«Обновить анализ»**
3. Новый Score появится через 60 секунд, дельта (+5 / −3) покажется рядом

**Сценарий 3 — Поделиться с командой:**
1. Кнопка **«Поделиться»** в шапке
2. Создаётся публичная ссылка (read-only)
3. Отправляете дизайнеру / маркетологу — они видят рекомендации без аккаунта

## 3.5. Ключевые функции
- **Общий Score 0-100** с дельтой к прошлому анализу
- **6 категорий** с раскрытием деталей
- **«Главный вывод»** — AI-summary одним абзацем
- **Рекомендации с приоритетами** — Срочно / Важно / На потом
- **Живые рейтинги карт** — Google / Яндекс / 2GIS
- **«Обновить анализ»** — повторный прогон
- **«Поделиться»** — публичная ссылка
- **«Экспорт в PDF»** — для отправки клиенту

## 3.6. Интеграции
- Все источники из «Нового анализа» (см. модуль 2)
- **Keys.so** — SEO-позиции в категории SEO

## 3.7. Роли и права
- **Владелец / Manager:** полный доступ + редактирование рекомендаций
- **Viewer:** только просмотр
- **По ссылке (share):** read-only, без возможности обновить

## 3.8. Типичные кейсы

**Кейс 1 — План работ маркетолога на месяц:** распечатать раздел «Рекомендации (Срочно + Важно)» → отдать SMM-щику.

**Кейс 2 — Аргумент для бюджета:** дельта Score после внедрения рекомендаций (Score был 56 → стал 72) — наглядный аргумент перед собственником.

**Кейс 3 — Презентация инвесторам:** экспорт PDF с дашбордом для pitch-deck.

# 4. Предыдущие анализы

## 4.1. Что это и зачем
Архив всех проведённых анализов компании со сравнением Score-ов и историей изменений.

## 4.2. Как зайти
- **Сайдбар:** **АНАЛИТИКА** → **Моя компания** → **Предыдущие анализы**

## 4.3. Главный экран
🖼 **[СКРИН: Лента анализов с дельтами]**

Список карточек, каждая — отдельный анализ:
- Дата
- Score (с дельтой относительно предыдущего)
- Краткое summary
- Кнопки «Открыть отчёт», «PDF»

Сверху — график изменения Score за период.

## 4.4. Сценарии использования

**Сценарий 1 — Сравнение «было / стало»:**
1. Откройте список
2. Выберите старый анализ (например, 3 месяца назад)
3. Кликните **«Сравнить с последним»**
4. Увидите side-by-side: что улучшилось, что упало

**Сценарий 2 — Export ежемесячного отчёта:**
1. Найдите анализ за нужный месяц
2. Нажмите **PDF** на карточке
3. Получаете PDF-отчёт для архива / клиента

## 4.5. Ключевые функции
- **Лента анализов** хронологически
- **Дельты Score** между соседними анализами
- **Сравнение двух отчётов** side-by-side
- **PDF-экспорт** любого исторического отчёта
- **График динамики** Score за всё время

## 4.6. Интеграции
- Внутреннее хранилище localStorage (`mr_analysis_history_${userId}`)

## 4.7. Роли и права
- **Владелец / Manager:** видят всю историю
- **Viewer:** последние 3 анализа

## 4.8. Типичные кейсы

**Кейс 1 — Доказательство роста для собственника:** «За полгода Score вырос с 42 до 78 благодаря этим действиям…»

**Кейс 2 — Анализ провала:** «После редизайна сайта Score упал — что именно сломалось» (сравнение до/после).

**Кейс 3 — Документация для KPI команды:** ежемесячный PDF в отчётность отделу маркетинга.

# 5. AI-инсайты

## 5.1. Что это и зачем
Список AI-сгенерированных инсайтов по компании — нестандартные наблюдения, которые не попадают в стандартные категории.

## 5.2. Как зайти
- **Сайдбар:** **АНАЛИТИКА** → **Моя компания** → **AI-инсайты**
- В сайдбаре отображается счётчик новых инсайтов (например: `AI-инсайты 7`)

## 5.3. Главный экран
🖼 **[СКРИН: Список инсайтов с тегами]**

Карточки инсайтов с тегами: **Возможность**, **Риск**, **Аномалия**, **Тренд**.

Каждая карточка:
- Заголовок (одно предложение-инсайт)
- Контекст: что заметил AI
- Рекомендация: что с этим делать
- Источник: на какие данные опирается

## 5.4. Сценарии использования

**Сценарий 1 — Утренний review:**
1. В сайдбаре видите счётчик `AI-инсайты 7` — новых
2. Открываете список
3. Прокручиваете — отмечаете 1-2 для проработки командой
4. Нажимаете **«Архивировать»** на остальных

**Сценарий 2 — Дополнить план:**
1. На карточке инсайта — кнопка **«Создать пост из инсайта»**
2. Переход в Контент-завод с готовой темой
3. Один клик — пост готов

## 5.5. Ключевые функции
- **Лента инсайтов** с тегами
- **Фильтр по тегам** — только возможности / только риски
- **«Архивировать»** — убрать с глаз
- **«Создать пост»** — прямой переход в Контент-завод

## 5.6. Интеграции
- **Claude Sonnet 4.6** — генерация инсайтов из данных анализа
- Внутренние модули — источники данных

## 5.7. Роли и права
- Все роли видят, только Владелец может архивировать

## 5.8. Типичные кейсы

**Кейс 1 — Инсайт «У 3 ваших конкурентов резкий рост upbeat-вакансий — расширяются»** → подготовка контр-стратегии.

**Кейс 2 — Инсайт «Сезонный пик трафика в марте — за 6 недель»** → запуск рекламной кампании заранее.

**Кейс 3 — Инсайт «Отзывы с упоминанием цены выросли на 40%»** → решение пересмотреть позиционирование.

# 6. AI Видимость

## 6.1. Что это и зачем
Проверка, как часто и в каком контексте AI-сервисы (ChatGPT, Claude, Gemini, YandexGPT) упоминают вашу компанию по релевантным запросам.

## 6.2. Как зайти
- **Сайдбар:** **АНАЛИТИКА** → **Моя компания** → **AI Видимость**

## 6.3. Главный экран
🖼 **[СКРИН: Таблица упоминаний по AI-сервисам]**

Таблица:
- Колонки: ChatGPT / Claude / Gemini / YandexGPT
- Строки: запросы по вашей нише
- В ячейках: упомянуты ли вы, в каком месте топа

Сверху — общий процент AI-видимости.

## 6.4. Сценарии использования

**Сценарий 1 — Первый прогон:**
1. Откройте раздел
2. Нажмите **«Сгенерировать запросы»** — AI составит 20 вопросов из вашей ниши

# 7. SWOT-анализ

## 7.1. Что это и зачем
Классический SWOT (Strengths, Weaknesses, Opportunities, Threats) + матрица стратегий TOWS — построен автоматически из данных анализа компании.

## 7.2. Как зайти
- **Сайдбар:** **АНАЛИТИКА** → **Моя компания** → **SWOT-анализ**

## 7.3. Главный экран
🖼 **[СКРИН: 4 SWOT-карточки + TOWS матрица]**

Четыре крупные карточки:
- 🟢 **Сильные стороны** — «Что даёт нам преимущество прямо сейчас»
- 🔴 **Слабые стороны** — «Что тормозит развитие»
- 🔵 **Возможности** — «Куда стоит расти в ближайшее время»
- 🟠 **Угрозы** — «Что может ударить, к чему готовиться»

Каждая карточка раскрывается с детальным разбором.

Ниже — **TOWS-матрица стратегий**:
- SO (Атака) — Силы × Возможности
- ST (Защита) — Силы × Угрозы
- WO (Рост) — Слабости × Возможности
- WT (Оборона) — Слабости × Угрозы

В конце — **Стратегический синтез** с одним главным выводом.

## 7.4. Сценарии использования

**Сценарий 1 — Первая генерация:**
1. Откройте SWOT
2. Нажмите **«Сгенерировать SWOT»** (60-90 секунд)
3. Прочитайте 4 раздела
4. Нажмите **«Сгенерировать TOWS»** для стратегических пересечений

**Сценарий 2 — Стратегическая сессия:**
1. Распечатайте отчёт через **PDF**
2. На совещании раздайте команде
3. По каждому квадранту обсуждайте 15 минут

## 7.5. Ключевые функции
- **Автогенерация 4 SWOT-разделов** из данных компании
- **TOWS-матрица** — отдельная генерация
- **Стратегический синтез** — один абзац с главным выводом
- **Сохранение между сессиями** — отчёт не теряется при переключении табов
- **Экспорт HTML и PDF**
- **«Перегенерировать»** — обновить с учётом изменений

## 7.6. Интеграции
- **Claude Sonnet 4.6** — генерация
- Внутренние данные: анализ компании, СММ, ЦА

## 7.7. Роли и права
- **Владелец / Manager:** генерируют и редактируют
- **Viewer:** только просматривают

## 7.8. Типичные кейсы

**Кейс 1 — Подготовка к pitch инвесторам:** SWOT-слайд в презентации — основа для блока «Рынок».

**Кейс 2 — Антикризисное планирование:** в WT-квадранте (оборона) видны 2 угрозы — команда строит контр-меры.

**Кейс 3 — Onboarding нового топ-менеджера:** прочитав SWOT, новый CMO понимает контекст бизнеса за 15 минут.

- Топ-5 худших — что переделать

### ROI калькулятор:
- Вход: затраты на контент (часы × ставка + рекламный бюджет)
- Выход: выручка от привлечённых лидов
- Расчёт окупаемости в % и в днях

## 19.4. Сценарии использования

**Сценарий 1 — Внести метрики поста:**
1. В «Создать пост» → откройте опубликованный пост
2. Внизу — кнопка **«Внести метрики»** (drop zone для скрина статистики)
3. AI распознаёт цифры → они появляются в аналитике

**Сценарий 2 — Посчитать ROI кампании:**
1. ROI калькулятор → введите затраты часов и бюджета
2. Введите выручку
3. Получаете ROI %, payback в днях

- **Trends:** Trend Hunter
- **Publishing:** Auto-Publisher

**Сценарий 2 — Разовый запуск:**
1. На карточке агента — **«Запустить сейчас»**

**Сценарий 3 — Inbox approvals:**
1. Auto-Publisher работает в режиме черновика
2. Готовые публикации попадают в **«Inbox»**
3. Вы одобряете перед публикацией

## 22.5. Ключевые функции
- 5+ агентов в разных категориях
